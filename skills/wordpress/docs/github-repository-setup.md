## Suggested Release Notes

```markdown
Initial public work-in-progress release of WordPress Plugin Dev Skill.

### Included

- Portable canonical Agent Skill at `skills/wordpress-plugin-dev/`
- Install-target copies for `.agents`, `.claude`, and `.cursor` workflows
- Curated references for WordPress plugin architecture, security, REST/admin, Gutenberg, Interactivity API, i18n/a11y/privacy, testing, CI, and WordPress.org release readiness
- Safe starter templates for plugin bootstrap, Composer, npm, block.json, REST controllers, settings pages, readme.txt, and GitHub Actions CI
- Heuristic audit script with human and JSON output
- Demo and sample fixtures for validation
- Compatibility notes for Codex, Claude Code, Cursor, and other Agent Skills-compatible tools

### Notes

- The audit script is heuristic and does not replace a professional security review.
- Cursor filesystem paths and slash invocation vary by version/workflow; verify current official docs or in-app support.
- This release is intentionally marked as work in progress.
```

## Recommended Repository Settings

- Enable issues.
- Create starter issues from `docs/starter-issues.md`.
- Pin a roadmap issue or enable discussions only if maintainers plan to monitor them.
- Enable private vulnerability reporting if available.
- Require pull requests before merging once more contributors are involved.
- Add branch protection after the validation workflow has run successfully at least once.
